Web app designed to extract very particular functionality from Notion and tweak in minor ways.

Homepage is, like Notion, text-enabled. The middle 1200px of the page is an infinite textbox.

Unlike Notion, every line is a todo by default. Each and every todo is saved with its datetime of creation. (This small tweak will enable very interesting features in the future.)

The goals of the app is

- make capture frictionless
- enable user to plan hierarchically
- allow for multiple levels of commitment/time-allocation
  - suppose for example we are planning a gym schedule. planning is hierarchical, occurs at multiple levels. we should have the ability to 'allocate' 7 hours/wk to the routine, to be reflected in the 'available hours' calculated attr for each week for the length of the gym routine commitment. (no commitments are permanent. every plan has an end date.) however when planning the week in front of us, those 7 hours are unallocated. they float in a "committed but unplanned" box outside the schedule-box, waiting to be dragged into a time-slot.
- extremely flexible, frictionless tagging of lines. certain keywords, if included in a line, should auto-tag that line as themselves
